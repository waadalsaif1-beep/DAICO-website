# دليل تشغيل الـ Backend وإعداد البريد الإلكتروني (DAICO Backend Server)

يوفر هذا المجلد كود الـ Backend الكامل لمنصة **دايكو (DAICO)** مكتوباً بلغتين مختلفتين لتختار منهما ما يناسبك:
1. **Node.js (Express)**
2. **Python (Flask)**

يقوم هذا الخادم بالوظائف التالية:
* تفعيل إرسال بريد إلكتروني حقيقي يحتوي على رمز الـ OTP المكون من 4 أرقام مباشرة إلى بريد المستخدم.
* تطبيق شروط وسياسة التحقق من قوة كلمة المرور على مستوى الـ Backend.
* توفير محاكاة برمجية كاملة لعمليات التسجيل، التفعيل، وتسجيل الدخول تمهيداً لعملية النقل للإنتاج (Production).

---

## 1. التكوين وإعداد بيئة العمل (Environment Configuration)

1. قم بنسخ ملف الإعدادات التجريبي `.env.example` وتسميته `.env`:
   ```bash
   cp .env.example .env
   ```
2. افتح ملف `.env` وقم بتهيئة إعدادات خدمة البريد الإلكتروني الخاصة بك (مثل SMTP أو SendGrid أو Mailgun أو AWS SES):

   ```env
   PORT=3000
   
   # إعدادات خدمة البريد الإلكتروني الحقيقية
   SMTP_HOST=smtp.sendgrid.net
   SMTP_PORT=587
   SMTP_SECURE=false
   SMTP_USER=apikey
   SMTP_PASS=SG.your_sendgrid_api_key_here
   SMTP_FROM="منصة دايكو" <info@daico.edu.sa>
   ```

*ملاحظة: إذا لم تقم بتكوين إعدادات البريد في ملف `.env`، سيقوم النظام تلقائياً بطباعة رمز الـ OTP في شاشة الـ Console الخاصة بالخادم وبثه محاكاةً لتسهيل التجربة دون إيقاف التشغيل.*

---

## 2. خيار تشغيل Node.js Backend (Express)

### المتطلبات
* تثبيت [Node.js](https://nodejs.org) (إصدار 16 أو أحدث).

### خطوات التشغيل
1. افتح مبدل الأوامر (Terminal) في هذا المجلد (`backend`):
   ```bash
   cd backend
   ```
2. قم بتثبيت الحزم المطلوبة:
   ```bash
   npm install
   ```
3. ابدأ تشغيل خادم الـ API:
   ```bash
   npm start
   ```

سيعمل الخادم على العنوان: `http://localhost:3000`.

---

## 3. خيار تشغيل Python Backend (Flask)

### المتطلبات
* تثبيت [Python](https://python.org) (إصدار 3.8 أو أحدث).

### خطوات التشغيل
1. افتح مبدل الأوامر (Terminal) في هذا المجلد (`backend`):
   ```bash
   cd backend
   ```
2. قم بإنشاء بيئة افتراضية (موصى به):
   ```bash
   python -m venv venv
   # لتفعيل البيئة في Windows:
   .\venv\Scripts\activate
   # لتفعيل البيئة في macOS/Linux:
   source venv/bin/activate
   ```
3. قم بتثبيت الحزم المطلوبة:
   ```bash
   pip install -r requirements.txt
   ```
4. ابدأ تشغيل الخادم:
   ```bash
   python app.py
   ```

سيعمل الخادم على العنوان: `http://localhost:3000`.

---

## 4. كيف تعمل عملية الدمج (Integration Architecture)

تمت برمجة النظام بأسلوب هجين ذكي (Smart Hybrid Architecture):
1. **قاعدة البيانات:** بما أن التطبيق الحالي هو نموذج مبدئي يعتمد بالكامل على المتصفح وتخزين `localStorage` (`db.js`) لتتبع المستخدمين والدورات والتحليلات، يتم حفظ بيانات الحساب الجديد محلياً في المتصفح للحفاظ على عمل لوحات التحكم (Admin/User Dashboard).
2. **إرسال الـ OTP الحقيقي:** بمجرد نجاح التسجيل محلياً، تقوم الواجهة الأمامية بإرسال طلب خلفي غير متزامن (Asynchronous Fetch) إلى خادم الـ Backend المفتوح على منفذ `3000` ليقوم بإرسال البريد الإلكتروني الحقيقي فوراً.
3. **أمان التشغيل والاستكشاف:**
   * إذا كان خادم الـ Backend **مفعلاً** وموصولاً ببيانات SMTP صحيحة، فسيصلك إيميل حقيقي فوراً بترميز HTML مميز ومصمم لعلامة دايكو.
   * إذا كان خادم الـ Backend **غير مفعّل** (مغلق)، فلن يتوقف الموقع عن العمل بل سيعطيك تحذيراً في الـ console بالمتصفح ويطبع لك الرمز هناك لتتمكن من نسخ الكود وإكمال تفعيل حسابك كمتصفح مستقل.
